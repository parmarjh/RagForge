◆ ALGORITHM

Step-by-Step Breakdown

Each stage in the RAG pipeline transforms data closer to a precise, source-backed answer.
01

Document Ingestion

Load content from multiple sources — PDF, DOCX, TXT, web pages, REST APIs, and SQL databases into unified Document objects.

02

Text Chunking

Split documents into optimal-sized chunks using recursive, fixed-size, or semantic strategies with configurable overlap for context preservation.

03

Embedding Generation

Convert text chunks into high-dimensional vectors using embedding models — OpenAI (text-embedding-3), HuggingFace (MiniLM), or Ollama (local).

04

Vector Storage

Store embeddings in Qdrant vector database with full metadata payload. Supports in-memory mode for development, persistent for production.

05

Similarity Search

Embed the user's query and find the most relevant chunks via cosine similarity in vector space. Configurable top-K and score thresholds.

06

Reranking & Generation

Optionally re-score results with an LLM for improved relevance, then feed top context + question to generate a synthesized, cited answer.

Installation & Usage

bash 
# Core (Qdrant + requests)
pip install ragforge
bash 
# OpenAI embeddings + LLM
pip install ragforge[openai]

# Anthropic Claude LLM
pip install ragforge[anthropic]

# Local sentence-transformers
pip install ragforge[huggingface]
bash 
# Everything included
pip install ragforge[all]
bash 
git clone https://github.com/parmarjh/RagForge.git
cd RagForge/ragforge
pip install -e .
python — SDK usage 
from ragforge import RAGEngine

engine = RAGEngine(
    embedding_provider="openai",
    llm_provider="anthropic",
    qdrant_url="http://localhost:6333",
)

# Ingest documents
engine.ingest_file("docs/manual.pdf")
engine.ingest_url("https://docs.python.org/3/")
engine.ingest_text("Inline text to index")

# Query with source attribution
result = engine.query("How do I handle exceptions?")
print(result.answer)
print(result.sources)
print(result.confidence)
◆ ARCHITECTURE

Package Structure

Module Purpose
ragforge.core Engine orchestrator, configuration dataclasses, shared types
ragforge.ingest Document loaders (PDF, DOCX, text, web, API, DB) + text chunker
ragforge.embeddings Embedding providers (OpenAI, HuggingFace, Ollama)
ragforge.vectorstore Vector store implementations (Qdrant with in-memory / persistent)
ragforge.llm LLM providers (OpenAI GPT, Anthropic Claude, Ollama)
ragforge.retrieval Retriever (vector search) and LLM-based reranker
ragforge.pipeline RAG pipeline orchestration and prompt templates
